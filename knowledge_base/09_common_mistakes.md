# Common Mistakes in Lease Accounting Extraction

## 1. Possession Date vs. Rent Commencement Date
- **Mistake:** Assuming the rent-free or fit-out period start date is the commencement date for accounting.
- **Correct Rule:** Under IND AS 116, the lease commencement date is when the lessor makes the asset available for use (usually possession date), NOT when rent starts. The rent-free period should be factored in as a lease payment waiver, and the lease term starts on the possession date.

## 2. Ignoring Escalation Clauses
- **Mistake:** Using a single flat rent amount throughout the lease term when the document contains escalation clauses (e.g. 5% increase annually).
- **Correct Rule:** Escalation clauses must be modeled either as systematic escalations (percentage increase on a specific date) or adhoc escalations (fixed price steps).

## 3. Disregarding Lock-in Period Defaults
- **Mistake:** Setting lock-in period to null when it is not explicitly mentioned.
- **Correct Rule:** If not explicitly stated, the lock-in period and lease working period default to the main lease period.

## 4. Incorrect Discounting
- **Mistake:** Discounting the security deposit as part of lease payments or ignoring it entirely.
- **Correct Rule:** Security deposits are discounted separately to calculate prepaid rent (asset) and are not included in the lease liability stream.
